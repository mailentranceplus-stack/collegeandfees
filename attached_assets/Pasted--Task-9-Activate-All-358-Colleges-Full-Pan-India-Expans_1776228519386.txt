# Task #9 — Activate All 358 Colleges (Full Pan-India Expansion)

> **Date:** 2026-04-15
> **Priority:** CRITICAL — 308 college pages are returning 404 despite having complete fee data + content in DB

---

## What & Why

The Supabase database now has **358 colleges** with:
- Complete fee data (9,025 fee rows across 28 branches and 7 quotas)
- `college_content` entries for ALL 358 (about, highlights, admission_process, meta_title, meta_desc)
- `is_active = true` on all 358

But the website only shows 50 colleges because `ACTIVE_SLUGS` in `lib/constants.js` is hardcoded to 50 slugs. The remaining 308 colleges — including IITs, NITs, IIITs, BITS Pilani, VIT, and 150+ Karnataka colleges — all return 404.

**SEO impact:** 308 colleges × 4 pages each = **1,232 missing indexable URLs**. These are high-value pages (e.g., "BITS Pilani fees", "IIT Bombay fees", "VIT Vellore fees") that would rank and drive traffic.

---

3. `/sitemap.xml` includes all 358 colleges (the sitemap already filters by `is_active` from DB — just confirm it picks up the new pages)
4. The college listing page at `/colleges` shows all 358 colleges
5. `scripts/audit.js` passes (update any hardcoded slug checks if needed)
6. No design changes whatsoever

Instead of hardcoding 358 slugs (which will grow), **consider replacing the hardcoded `ACTIVE_SLUGS` Set with a DB query.** The `colleges` table already has `is_active = true` for all active colleges. The recommended approach:

**Option A (Preferred):** In `getServerSideProps` for college pages, query Supabase for the college by slug where `is_active = true`. If found, render. If not, 404. Remove the `ACTIVE_SLUGS` check entirely.

**Option B (If Option A is too risky):** Replace `ACTIVE_SLUGS` with the full 358-slug Set below.

If you go with Option A, also update:
- The sitemap generation to query all `is_active = true` colleges from DB

- Any other place that references `ACTIVE_SLUGS`
- Remove or comment out the `ACTIVE_SLUGS` export
