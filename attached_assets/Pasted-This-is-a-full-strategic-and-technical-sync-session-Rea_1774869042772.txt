THE CONTENT WORKFLOW — CRITICAL TO UNDERSTAND
There are two systems working on this project in parallel:
1. YOU (Replit) — own the code. Build pages, templates, components, routing, 
   JSON-LD, sitemaps, performance. Never touch content.
2. CoWork (a separate AI assistant with Supabase access) — owns the data. 
   Writes and manages all text content, fees data, FAQ answers, about text, 
   admission process descriptions, meta descriptions for college pages. 
   Pushes everything directly to Supabase. Never touches code.

When CoWork updates content in Supabase, the next page load on the live site 
shows the new content automatically because of SSR (getServerSideProps). No 
code deployment needed for content changes. This is the entire reason SSR 
was chosen over static generation.
